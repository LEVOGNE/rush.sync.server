Features

Everything built in

⚙

Reverse Proxy

Subdomain-based routing with automatic service discovery. Each server gets its own subdomain, TLS certificate, and isolated environment.

⚡

Multi-Server

Run up to 50 independent web servers simultaneously. Each with its own port, directory, logs, and configuration. Create, start, stop individually.

🔒

Auto-TLS

Automatic certificate generation per server and proxy. Self-signed for development, Let's Encrypt ready for production. Zero manual setup.

🔑

API-Key Authentication

HMAC-SHA256 hashed keys with timing-safe comparison. Environment variable support. Rate limiting per IP on all API endpoints.

🔄

Hot Reload

File system watcher detects changes and pushes updates via WebSocket. Edit HTML/CSS/JS and see results instantly without manual refresh.

📊

Logging & Monitoring

Per-server log files with rotation, GZIP compression, and configurable retention. Request logging, security alerts, and performance metrics.

💾

Persistence

Server registry survives restarts. Auto-start marked servers on boot. Perfect for Docker containers and headless deployments.

🌐

HTTP → HTTPS Redirect

Automatic redirect server on port 80. All HTTP traffic is seamlessly upgraded to HTTPS. Configurable target port.

💻

Terminal UI & Headless

Interactive TUI with themes, command history, and i18n (EN/DE). Or run as headless daemon for Docker and production servers.

Security

Hardened by default

HMAC-SHA256

API keys are hashed, never stored in plaintext. Generate with --hash-key

Timing-Safe Compare

Constant-time key comparison prevents timing side-channel attacks.

Rate Limiting

Configurable per-IP rate limiting on all /api/* endpoints. Default: 100 req/s.

Path Traversal Guard

Middleware blocks directory traversal, null bytes, and suspicious request patterns.

Roadmap

The vision: Self-Hosted PaaS

Rush Sync Server is more than a web server — it's the foundation for a complete, self-hosted Platform as a Service. The vision: 100+ servers with full orchestration on a single machine, with absolute perfect performance. One Rust binary that replaces your entire infrastructure stack. No vendor lock-in. No monthly cloud bills. Full control.

PHASE 1 Data Layer
🗃

SQLite Integration

Persistent storage per server — user sessions, analytics, form submissions, key-value store. Zero-config, embedded, serverless.

🗂

PostgreSQL Support

Production-scale relational database with connection pooling, migrations, and multi-tenant isolation for large deployments.

⚡

Built-in KV Store

Fast in-memory cache with disk persistence — like Redis, but built into the binary. No external services required.

PHASE 2 Dynamic Server
🔨

Embedded Scripting (Rhai/Lua)

Server-side scripting directly inside the server. Define custom API routes, business logic, and dynamic pages — no external runtime needed.

⚡

WASM Runtime

Execute WebAssembly modules server-side with near-native speed. Write server logic in Rust, Go, C, or any language that compiles to WASM. Sandboxed, secure, blazing fast.

⏰

Cron & Task Scheduler

Schedule recurring jobs — automated backups, database cleanup, webhook triggers, report generation. All built-in.

📩

Form Handler & Webhooks

Process HTML forms natively, store in DB, send email notifications. Incoming/outgoing webhooks with retry logic and event queuing.

📄

Template Engine (SSR)

Server-Side Rendering with Tera/Handlebars templates. Dynamic pages, layouts, partials — full SSR without a JavaScript framework.

PHASE 3 Platform Features
👥

User Management & Auth

Built-in registration, login, JWT tokens, OAuth2 provider, and role-based access control (RBAC). Multi-tenant ready.

📈

Real-Time Analytics

Page views, unique visitors, geolocation, error tracking, performance profiling — all built-in, no third-party scripts.

📦

Asset Pipeline

CSS/JS minification, image optimization, Gzip/Brotli pre-compression. Automatic cache-busting with content hashes.

🔌

Git Deploy

Push-to-deploy via built-in Git server or webhook. Automatic builds on git push — your own mini CI/CD.

💾

Automated Backups

Scheduled snapshots of databases, configs, and files. S3-compatible upload, encrypted storage, one-click restore.

PHASE 4 The Future
🌎

Edge Functions

Execute code at the edge — Cloudflare Workers-style, but self-hosted. WASM-powered, millisecond cold starts.

🔗

Multi-Node Clustering

Distribute workloads across multiple servers. Load balancing, failover, zero-downtime deployments.

🔌

Plugin System (WASM)

Community-extensible via WebAssembly plugins. Sandboxed, secure, language-agnostic. Build and share extensions.

💻

Web IDE

Edit code directly in the browser. Syntax highlighting, file explorer, terminal — deploy without leaving the dashboard.

✉

Transactional Email

Built-in SMTP server for sending emails directly from your applications. Templates, tracking, bounce handling.
